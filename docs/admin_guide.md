# 平台管理員操作手冊 (Admin Guide)

本手冊旨在協助平台管理員有效地維護「南區綠活圖」的資料品質，確保資訊的準確性與即時性。

## 1. 如何取得管理員權限

本平台的管理員權限是透過後端資料庫手動設定的，以確保最高的安全性。

1.  **提供您的 UID**：
    - 使用您的 Google 帳號正常登入一次平台。
    - 聯絡系統的超級管理員，並提供您的 Firebase 使用者 UID (User ID)。
2.  **等待授權**：超級管理員會將您的 UID 添加到 Firestore 資料庫的 `admins` 集合中。
3.  **完成**：一旦添加完成，您下次登入時，系統將自動賦予您管理員權限。

## 2. 進入管理後台

擁有管理員權限的使用者，可以直接透過 URL 進入後台。

- **網址**：在瀏覽器中手動輸入 `您的網域/admin`。
- **驗證**：系統會驗證您的登入狀態與管理員身份。如果驗證通過，您將看到管理後台介面；否則將被導回首頁。

## 3. 管理後台功能

後台主要分為兩大功能區塊，您可以透過左側的導覽列進行切換。

### 待審核地點 (F-C1)

此頁面列出了所有由「綠活伙伴」提交，正在等待審核的新地點。

- **審核項目**：每個項目都會顯示地點的名稱、地址、描述以及一張預覽照片的連結。
- **操作**：
  - **核准 (Approve)**：
    - 點擊「核准」後，該地點的資料將被**正式發布**到公開地圖上。
    - 該筆申請會從待審核列表中移除。
  - **拒絕 (Reject)**：
    - 點擊「拒絕」後，該筆申請資料（包含上傳的照片）將被**永久刪除**。
    - 該筆申請會從待審核列表中移除。

### 錯誤回報 (F-C2)

此頁面列出了所有由使用者提交的地點錯誤回報。

- **篩選**：您可以點擊「待處理」或「已解決」來篩選不同狀態的回報。
- **審核項目**：每個項目會顯示被回報的地點名稱、使用者填寫的問題描述以及回報時間。
- **操作**：
  - **標示為已解決 (Mark as Resolved)**：
    - 當您確認問題已處理完畢（例如，已手動修正該地點的錯誤資訊）後，點擊此按鈕。
    - 該筆回報的狀態會變為 `resolved`，並從「待處理」列表中移除。這有助於保留處理紀錄。

---

請謹慎操作，您的每一次點擊都將直接影響公開地圖的內容。感謝您的貢獻！
